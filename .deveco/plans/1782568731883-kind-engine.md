# 导入重写 + 批量管理补全计划

## 1. ImportSourceDialog 完全重写

参照 Android `ImportBookSourceDialog.kt`，实现预览式导入：

### UI 结构
- **顶部**：关闭按钮 + 标题 "导入书源 (X条)"
- **预览列表**：每行显示书源名、URL、状态标签（新增🟢/更新🟠/已有⚪）+ Checkbox
- **快捷操作**：全选/取消全选、仅选新增、仅选更新
- **选项区**：
  - 保留原名（Toggle，默认true）
  - 保留分组（Toggle，默认true）  
  - 保留启用（Toggle，默认true）
  - 自定义分组（TextInput + 追加/覆盖切换）
- **底部**：导入确认按钮 + "已选 X/Y 项" 计数

### 数据流
1. 解析 JSON → 对比数据库 → 生成 ImportSourcePreview[]（含状态标记）
2. 用户勾选 + 设置选项
3. 确认 → `dao.importSelectedSources(previews, options)` → 写入DB

### BookSourceTable 新增方法
- `importSourcesPreview(json): ImportSourcePreview[]` — 解析+对比，返回预览列表
- `importSelectedSources(previews, options): number` — 按选项导入选中书源

## 2. BookSourcePage 批量管理补全

- **批量分组**：在底部操作栏加「分组」按钮 → 弹出分组选择器
- **全选/反选**：底部操作栏加「全选」按钮
- **校验入口**：底部加「校验」按钮 → 调用 SourceChecker
- **编辑入口**：列表项长按菜单/点击进入 BookSourceEditPage
