## 改造计划：支持 `{{expression}}` 模板表达式求值

### 问题分析
`searchUrl` 模板 `{{Get('url')}}/search/photos?search_query={{key}}&search-type=photos&main_tag=0&page={{page}}` 中的 `{{Get('url')}}` 在当前代码中直接被正则 `/\{\{[^}]*\}\}/g` 剥离，导致搜索 URL 退回到 `sourceUrl`。

### 改造方案（3 个文件修改）

---

### 1. `JsExpressionEvaluator.ts` — 添加 `source.getVariable()` 支持

**文件**: `entry/src/main/ets/engine/source/JsExpressionEvaluator.ts`

**a) `JsEvalContext` 接口新增字段** (line ~71):
```typescript
export interface JsEvalContext {
  key?: string;
  page?: number;
  baseUrl?: string;
  source?: Partial<BookSource>;
  jsLib?: string;
  /** 书源变量的 JSON 字符串（用于 source.getVariable/setVariable 持久化） */
  variableBlob?: string;
  [key: string]: unknown;
}
```

**b) `buildContextScript` 中给 `source` 添加 `getVariable`/`setVariable`** (line ~479，在 `getKey`/`getUrl` 之后):
```typescript
// source.getVariable() / setVariable() — 书源变量持久化
parts.push(`(function(){var _vars=${JSON.stringify(ctx.variableBlob || '{}')};` +
  `source.getVariable=function(){return _vars;};` +
  `source.setVariable=function(v){_vars=v;};` +
  `})();`);
```

初始值来自 `ctx.variableBlob`，默认为 `{}`。调用方负责传入初始变量 JSON。

### 2. `SourceExecutor.ts` — 在 `searchSingle` 中评估 `{{expression}}` 模式

**文件**: `entry/src/main/ets/engine/source/SourceExecutor.ts`

**在 `<js>` 块求值之后（line ~570）、`buildUrl` 调用之前（line ~572）**，插入自定义 `{{}}` 表达式求值：

```typescript
// === 新增：评估 {{expression}} 模式 ===
searchUrlTemplate = await this.evaluateTemplateExpressions(
  searchUrlTemplate, source, keyword, page, baseUrl
);
```

**新增方法 `evaluateTemplateExpressions`**（放在 `searchSingle` 之后或作为独立方法）:

```typescript
/**
 * 评估 searchUrl 模板中未被 replaceSearchTemplateVars 处理的 {{expression}} 模式
 * 如 {{Get('url')}}、{{Get('shunt')}} 等 JS 函数调用
 */
private async evaluateTemplateExpressions(
  template: string, source: BookSource,
  keyword: string, page: number, baseUrl: string
): Promise<string> {
  if (!template || !this.engineInitialized) return template;

  // 构建 JS 上下文：source + jsLib + 基础变量
  const ctx: JsEvalContext = {
    key: keyword,
    page: page,
    baseUrl: baseUrl,
    source: source,
    jsLib: source.jsLib || '',
    variableBlob: source.variableComment || '',
  };

  // 匹配所有 {{...}} 模式，排除已知的简单变量模式
  const templateVarPattern = /\{\{(key|keyword|page|pageNum)\}\}|\{\{(page|pageNum)\s*[+-]\s*\d+\s*\}\}/gi;
  
  return template.replace(/\{\{([^}]+)\}\}/g, (match, expr: string): string => {
    // 跳过已知的简单变量（由 replaceSearchTemplateVars 处理）
    if (templateVarPattern.test(match)) return match;
    
    // 尝试作为 JS 表达式求值
    try {
      const result = await JsExpressionEvaluator.evaluate(expr, ctx);
      if (result && result.trim() && result !== 'null' && result !== 'undefined') {
        return result.trim();
      }
    } catch (_) { /* 求值失败，保留原样让 buildUrl 处理 */ }
    return match; // 保留原样，buildUrl 后续会剥离
  });
}
```

**关键设计点**:
- 用 `templateVarPattern` 识别已知变量模式并跳过（它们由 `replaceSearchTemplateVars` 处理）
- 对未知 `{{}}` 模式尝试 JS 求值
- 如果求值失败，保留原模式字符串，让 `buildUrl` 后续剥离（兼容现有行为）
- `this.engineInitialized` 检查避免在引擎未初始化时调用

### 3. `SourceExecutor.ts` — `searchUrl 调用的 source.variableComment` 传入

在 `searchSingle` 中调用 `evaluateTemplateExpressions` 时，从 `source.variableComment` 获取初始变量 JSON。

```typescript
// searchSingle 中 (在调用 evaluateTemplateExpressions 时)：
const initialVariables = source.variableComment || source.rawJson || '';
searchUrlTemplate = await this.evaluateTemplateExpressions(
  searchUrlTemplate, source, keyword, page, baseUrl
);
```

### 影响范围
| 文件 | 改动 |
|------|------|
| `JsExpressionEvaluator.ts` | `JsEvalContext` 接口 + `buildContextScript` 中各加数行 |
| `SourceExecutor.ts` | `searchSingle` 中插入约 2 行调用 + 新增约 30 行的 `evaluateTemplateExpressions` 方法 |

### 不涉及的改动
- `buildUrl` 函数不改动（`replaceSearchTemplateVars` 和 `{{}}` 剥离逻辑保留不变）
- `ExplorePage` 等探索页的 JS 求值不改动（本次仅聚焦搜索）
- 数据库持久化不改动（`getVariable`/`setVariable` 使用内存 + `variableComment` 字段初始值）
