# 新增"单页全屏"翻页模式

## 问题
当前翻页模式（左->右/右->左）中 Image 使用 `width('100%')` + `objectFit(ImageFit.Auto)`，图片宽度撑满但高度不限制，竖长图会超出屏幕、宽图在宽屏手机上偏小。PuraX 等宽屏手机显示效果不佳。

## 方案
新增第 4 种阅读模式 `READ_MODE_SINGLE_PAGE = 3`（单页全屏），每屏完整显示一张图片，图片用 `ImageFit.Contain` 按比例缩放到屏幕内（不裁切、不溢出），居中显示。

## 修改内容

### 1. ComicReadPage.ets
- 新增常量 `READ_MODE_SINGLE_PAGE = 3`
- 新增 `buildSinglePageMode()` Builder：
  - 使用 Swiper 翻页
  - 每个 Stack 充满全屏（`width('100%').height('100%')`）
  - Image 使用 `width('100%').height('100%')` + `objectFit(ImageFit.Contain)` 
  - 居中对齐（Stack 默认居中）
  - 保留手势（PinchGesture + 双击缩放）
- `build()` 中增加判断：`readMode === READ_MODE_SINGLE_PAGE` -> `buildSinglePageMode()`
- `readModeLabel()` 增加 `SINGLE_PAGE` -> '单页'
- `getDisplayImageUrls()`：单页模式不需要 RTL 反转
- `nextPage()`/`prevPage()`：单页模式走 Swiper 翻页逻辑
- `loadChapterImages` 恢复位置：单页模式用 `swiperController_.changeIndex`
- 模式选择面板 `buildModeSelectPanel` 新增选项
- `dispatchZoneAction_` 中的翻页逻辑覆盖单页模式

### 2. SettingsPage.ets
- `buildComicSettings` 阅读方向选项新增"单页全屏"

### 3. SettingsStore.ts
- 无需修改（`comic_read_mode` 已支持任意 int 值，默认 0）

## 验证
- 编译通过
- 单页模式下每张图完整居中显示在屏幕内
- 左右翻页正常切换图片
- 触控区域翻页正常
- 模式选择面板和设置页有新选项
